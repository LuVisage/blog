## 为什么做这个工具？

微调大模型这件事，对新手来说坑实在太多了——显存要多少才够？Rank 和 Alpha 设多大？学习率怎么选？好不容易把参数调完，训练脚本还得自己从头写。

每次做 LoRA 微调都要在这些琐碎问题上反复折腾，于是我把自己积累的脚本和经验打包成了一个完整的工具——**lora-trainer**。

上传你的 JSONL 数据，它会帮你分析数据质量、推荐超参数、估算显存占用，最后生成一套可直接运行的训练脚本。从数据到开始训练，四步走完，一气呵成。

## 它能做什么？

lora-trainer 把 LoRA 微调的完整流程拆成了四个步骤，每一步都有对应的工具：

### 1. 数据分析

拖入 JSONL 文件，自动识别数据格式（instruction-output / messages / conversations），统计长度分布，估算 Token 用量，同时检查空回复、重复数据等质量问题，帮你第一时间发现数据中的坑。

```
总样本数: 2340
数据格式: instruction-output
P95 Token 估算: 149  →  建议 max_seq_length ≥ 149
```

### 2. 超参数推荐

根据你的数据量、模型大小、任务类型和可用显存，智能推荐 Rank、Alpha、学习率、Epochs、Batch Size 等全部关键超参数，再也不用对着论文猜参数了。

以 2340 条对话数据 + Qwen2-7B + 24GB 显存为例：

| 参数 | 推荐值 | 理由 |
|------|--------|------|
| Rank (r) | 8 | 中等数据集，r=8 平衡学习能力与泛化 |
| 学习率 | 2e-4 | 标准 rank 搭配标准学习率 |
| Epochs | 3 | 中等数据集，3 轮充分收敛 |

内置 40+ 模型的规格数据库，覆盖 Qwen2、LLaMA 3、Mistral、DeepSeek、ChatGLM 等 12 个主流系列。

### 3. 显存估算

选好模型和参数，精确估算训练所需的 GPU 显存——模型权重、激活值、优化器状态、额外开销，每一项都算得明明白白，不会再出现"显卡炸了"的尴尬。

```
QLoRA 4-bit · qwen2-7b · seq=2048 · bs=4 · r=8

预估总显存: 4.63 GB — ✅ 轻松运行
```

### 4. 一键生成脚本

确认所有参数后，一键生成完整训练脚本（QLoRA + Flash Attention 2 + NEFTune），附带推理脚本和评估脚本，打包下载 ZIP。

生成的代码干净利落，改两行路径就能直接跑：

```python
MODEL_NAME = "Qwen/Qwen2-7B"   # 改成你的模型路径
DATA_PATH = "./my_data.jsonl"   # 改成你的数据路径
```

## 技术栈

整个项目以三种形态提供，按需取用：

- **CLI 工具** — `pip install lora-trainer`，命令行一键调用
- **Claude Code 插件** — 在 Claude Code 里输入 `/lora:cook` 即可触发
- **Web 界面** — 纯前端页面，浏览器打开即用，无需安装任何东西

核心推荐算法基于 LoRA 论文和社区实践经验，所有计算逻辑（Token 估算、显存计算、参数推荐）全部开源，不依赖任何外部 API。

## 快速开始

**命令行：**

```bash
pip install lora-trainer
lora-trainer cook --data my_data.jsonl --model qwen2-7b --task chat
```

**Web 在线使用：**

打开浏览器上传数据就能用，零安装成本。

**Claude Code：**

```bash
npx skills add LuVisage/lora-skills
# 然后在对话中输入 /lora:cook
```
